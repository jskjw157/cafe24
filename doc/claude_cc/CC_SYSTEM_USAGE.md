# `cc-system` 사용 가이드

이 프로젝트는 **Claude Code 환경을 커스터마이징하고 기능을 확장하기 위한 리소스 저장소**입니다. "Skills"를 통해 AI의 기능을 확장하고, 프롬프트를 최적화하며, 복잡한 에이전트 워크플로우를 설계할 수 있는 프레임워크를 제공합니다.

## 1. 프로젝트 개요

-   **핵심 기능**: Claude Code 환경의 기능 향상 및 맞춤 설정.
-   **주요 구성 요소**:
    -   **Skills 프레임워크** (`.claude/skills/`): AI가 필요에 따라 불러올 수 있는 모듈식 기능(Skills) 정의.
    -   **프롬프트 엔지니어링** (`prompt/`): 고효율 프롬프트 작성을 위한 템플릿.
    -   **문서** (`docs/cc/`): 서브 에이전트(Sub-agents), 훅(Hooks), 슬래시 명령어(Slash Commands)와 같은 고급 기능 가이드.

## 2. 주요 기능 및 사용법

### 2.1 새로운 스킬(Skill) 만들기
이 프로젝트에는 다른 스킬을 만드는 데 도움을 주는 `skill-creator` 스킬(`.claude/skills/skill-creator` 위치)이 포함되어 있습니다.

**사용 방법:**
1.  **참조**: `.claude/skills/skill-creator/SKILL.md`를 읽어 스킬의 구조를 파악합니다.
2.  **구조 생성**: `.claude/skills/<새로운-스킬-이름>` 경로에 새 디렉토리를 만듭니다.
3.  **파일 구성**:
    -   `SKILL.md` (필수): 스킬의 이름, 설명(트리거 조건), 지침을 정의합니다.
    -   `scripts/` (선택 사항): 결정론적 작업을 위한 Python/Bash 스크립트.
    -   `references/` (선택 사항): 문서화 파일(API 문서, 스키마 등)을 위한 Markdown 파일.
    -   `assets/` (선택 사항): 템플릿이나 정적 파일.
4.  **원칙**: "점진적 공개(Progressive Disclosure)" 원칙을 따르세요. 메인 `SKILL.md`는 가볍게 유지하고, 무거운 문서는 필요할 때만 참조(references)를 통해 불러오도록 합니다.

### 2.2 프롬프트 최적화
`prompt/crystalize-prompt.md` 템플릿을 사용하여 길고 복잡한 지침을 고효율의 토큰으로 압축하세요. 이는 문맥(Context) 윈도우 공간을 절약하면서도 의도를 유지하는 데 도움이 됩니다.

**사용 방법:**
1.  기존의 긴 시스템 프롬프트나 지침을 `crystalize-prompt.md`의 "입력(Input)" 섹션에 붙여넣습니다.
2.  압축 작업을 실행합니다 (Claude에게 텍스트에 "crystallization prompt를 적용해줘"라고 요청).
3.  결과로 나온 "압축된 프롬프트"를 에이전트 설정에 사용합니다.

### 2.3 에이전트 파이프라인 설계
`prompt/design-pipeline.md` 파일은 다중 에이전트 시스템을 설계하기 위한 로드맵을 제공합니다. 특히 "마스터 에이전트(Master Agent)"와 "서브 에이전트(Sub-agents)" 구조를 언급합니다.

**사용 방법:**
1.  복잡한 새 기능(예: "이 Framer 페이지를 React로 클론 코딩해줘")을 계획할 때 이 파일을 체크리스트/템플릿으로 사용하세요.
2.  다음 사항들을 정의합니다:
    -   **데이터 처리**: 전처리(Pre-processing)/후처리(Post-processing) 단계.
    -   **에이전트 역할**: 책임 분배 (마스터 vs 서브 에이전트).
    -   **문맥(Context) 관리**: 문맥 사용량을 낮게 유지하기 위한 전략.

## 3. 고급 설정 (문서)

`docs/cc` 폴더에는 환경 구성을 위한 유용한 정보가 들어 있습니다:

-   **서브 에이전트 (Sub-agents)** (`docs/cc/sub-agent.md`):
    -   `.claude/agents/` 또는 `~/.claude/agents/`에 특수 목적의 에이전트를 정의하는 방법을 배웁니다.
    -   예시: 특정 지침과 도구를 가진 "코드 리뷰어" 에이전트 생성.
-   **훅(Hooks) & 명령어**: (파일 이름으로 유추) 훅을 사용하여 작업을 자동화하거나 사용자 정의 슬래시 명령어를 만드는 방법을 배웁니다.

## 요약: 실행 가능한 다음 단계
1.  **`skill-creator` 탐구**: `SKILL.md` 가이드를 따라 간단한 스킬(예: "문서 도우미" 또는 특정 프로젝트 워크플로우)을 만들어 보세요.
2.  **문맥 압축**: 긴 문서나 프롬프트가 있다면 `crystalize-prompt.md`를 사용하여 최적화하세요.
3.  **서브 에이전트 설정**: `docs/cc/sub-agent.md`를 확인하고 자주 처리하는 반복 작업을 위한 사용자 정의 서브 에이전트를 정의해 보세요.
